Sanitize When Using LLMs

Safer to work locally than to upload real student data to cloud AI services.

  • 01Remove names, emails, student IDs, and essay text.
  • 02Blur or crop LMS screenshots — no gradebook faces or IDs.
  • 03Never paste FERPA / PII into public chat tools.
  • 04Work from a sanitized copy — keep raw files offline.
  • 05Use structure & aggregates — objectives, rubric criteria, point weights.
  • 06Today's demos use fake data — swap in your sanitized export when you go live.

> Local tools first. Cloud LLMs only with sanitized structure — never raw student work.

Fix It With the Bot

Something broke? The error message is the prompt. Feed it back to the agent.

  • 1Reproduce — note what you clicked and what should have happened.
  • 2Open the Console — browser DevTools → Console tab (F12 or right-click → Inspect).
  • 3Copy the red error — verbatim, including file name and line number.
  • 4Paste into Cursor — add context: what you tried, what you expected, which file.
  • 5Apply · refresh · retry — loop until the console is clean.
