Vibe Coding Cheat Sheet

Vocab from the workshop, core principles, and how to get help from the bot when something breaks.

Vocab we learned

Vibe coding
Describe what you want in plain English; the AI writes the code. You stay the teacher — and the judge.
Canvas / Artifacts
ChatGPT Canvas, Claude Artifacts, Gemini Canvas — in-browser spaces where the model drafts HTML, apps, and docs in one shot.
Sanitize
Strip names, essays, IDs, and gradebook faces before any data touches a public AI tool. Use structure & aggregates only.
Fake data
Made-up demo content for building and presenting — swap in sanitized exports when you go live.
One-off HTML
A single file that opens in any browser — no backend, no server, no install.
Vanilla JS
Plain JavaScript without a framework — or add a CDN library (Chart.js, Leaflet) when you need it.
Teacher-readable labels
Plain English on every screen so you (and students) know what each part does.
Static link
A URL that serves your HTML/JS/CSS as files — no server code required.
Deploy
Publish your project so anyone with a link can open it in a browser.
IDE
Integrated development environment — Cursor is an IDE with a built-in AI agent that sees your whole project.
Repo / multi-file project
HTML + data + styles as separate files you version, iterate, and deploy together.
Handoff
Export from Claude Design (or similar) into a code workspace to refine in an IDE.
Vibe-rary
Card catalog of teacher-made builds — borrow prompts, remix for your unit.
Console
Browser DevTools tab (F12) where JavaScript errors appear in red — copy them verbatim for the bot.

Core principles

You write requests, not syntax. The machine handles code; you handle pedagogy, fairness, and taste.
Sanitize before you prompt. Local tools first when real student data is involved; cloud LLMs only with sanitized structure.
Steal the pattern: DATA → VIEWS → STYLE → BUILD → LIBS → SHARE. Fill in your unit; reuse the skeleton.
Start one-off, level up when needed. Email a file → Netlify/Vercel/Lovable → GitHub Pages → Cursor for multi-file work.
Compare tools. Same prompt in ChatGPT, Claude, and Gemini — notice what each does differently.
Iterate conversationally. Build, test, paste errors back, refine — vibe coding is a loop, not a single shot.
Borrow freely, contribute back. The Vibe-rary and Co-Lab exist so teachers don't start alone.

Getting help from the bot

01Reproduce the bug — what did you click? What should have happened?
02Open DevTools → Console (F12 or right-click → Inspect). Red text = the clue.
03Copy the full error — message, file name, line number. Don't paraphrase.
04Paste into Cursor (or your chat tool) with context: what you tried, what you expected, which file.
05Apply the fix, refresh, retry. Loop until the console is clean.
example prompt
When I click Start I get this console error:

Uncaught TypeError: Cannot read properties of null
  (reading 'addEventListener') at game.js:47

Expected: board renders. Please fix game.js — vanilla JS only.

Avoid “it doesn't work.” The bot debugs better with exact errors + your intent. Same approach works for deploy steps — paste the terminal output.
